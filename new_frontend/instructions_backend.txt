## REQUIRED FEATURES

1. Voice Activity Detection

Use Silero VAD.

Requirements:

* Detect speech start
* Detect speech end
* Minimize unnecessary STT usage
* Fast turn-taking

2. Interruption Handling (Barge-In)

If the agent is speaking and the user starts talking:

* Immediately stop TTS playback
* Cancel current generation
* Switch to listening mode
* Process the new user instruction

Behavior should feel natural and conversational.

3. Conversation Memory

Maintain conversation history:

* system
* user
* assistant

Implement rolling history with configurable limit.

Store:

[
{
"role": "system",
"content": "..."
},
{
"role": "user",
"content": "..."
}
]

History should be automatically supplied to Groq.

4. Frontend Synchronization

The React frontend must be able to display:

* User messages
* Assistant messages
* Agent state

Use LiveKit Data Channels.

Send structured events:

{
"type": "user_message",
"content": "..."
}

{
"type": "assistant_message",
"content": "..."
}

{
"type": "agent_state",
"state": "idle"
}

{
"type": "agent_state",
"state": "listening"
}

{
"type": "agent_state",
"state": "thinking"
}

{
"type": "agent_state",
"state": "speaking"
}

{
"type": "agent_state",
"state": "interrupted"
}

5. Agent State Machine

Implement:

IDLE
LISTENING
THINKING
SPEAKING
INTERRUPTED

Provide clean state transitions.

6. Structured Logging

Log:

* User transcripts
* Assistant responses
* State transitions
* Interruptions
* Errors

Use Python logging.

7. Error Handling

Handle:

* Missing API keys
* Deepgram failures
* Groq failures
* Network errors
* Invalid room connections
* Deployment issues
